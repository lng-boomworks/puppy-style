# UX/UI Design References

These files contain design principles and patterns that the **design agent** must read before building any prospect website. They exist to ensure every site looks unique and industry-appropriate, not like a generic template with different colours.

## Files

| File | Purpose | Read when... |
|------|---------|--------------|
| `hero-patterns.md` | 6 hero layout patterns with decision matrix | Choosing the hero section layout |
| `section-rhythm.md` | Section ordering, spacing, background strategy | Planning the page structure |
| `typography-and-imagery.md` | Typography expression, image strategy, no-image fallbacks | Making typography and visual decisions |
| `cta-and-social-proof.md` | CTA design patterns, social proof placement, copy principles | Designing CTAs and trust elements |
| `industry-layouts.md` | Industry-specific layout guidance for 6 SMB verticals | Building for a specific business type |

## How the Design Agent Should Use These

1. **Before starting any build**, read `industry-layouts.md` to find the matching industry section
2. Use the hero pattern decision matrix in `hero-patterns.md` to pick a hero style
3. Plan the section order using `section-rhythm.md` — do NOT use the default template order
4. Apply typography and imagery decisions from `typography-and-imagery.md`
5. Design CTAs and social proof using `cta-and-social-proof.md`

## The "No Two Alike" Rule

Before deploying, verify these three things differ from the last site built:
1. **Hero pattern** — different layout type
2. **Section order** — different sequence of sections
3. **Background rhythm** — different light/dark/accent pattern

If any two consecutive sites share all three, the design is too similar.
